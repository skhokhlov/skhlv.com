## What this is

Personal website at https://skhlv.com — a Hugo static site deployed via GitHub Pages (see `CNAME`). The theme is [hugo-coder](https://github.com/luizdepra/hugo-coder), pulled in as a git submodule at `themes/hugo-coder`.

## Commands

- `hugo server -D` — local dev server with drafts, live reload at http://localhost:1313
- `hugo` — build the site into `public/` (the deployable output)
- `hugo new content/posts/<slug>.md` — scaffold new content from `archetypes/default.md`
- After cloning fresh: `git submodule update --init --recursive` to pull the theme

## Architecture notes

- Site config is `hugo.toml` (Hugo's newer format — not `config.toml`). It is intentionally minimal: only `baseURL`, `locale`, `title`, `theme`. All rendering behavior comes from the theme defaults.
- `content/`, `layouts/`, `static/`, `data/`, `i18n/` exist but are empty — the site currently renders entirely from theme defaults. To override theme behavior, mirror the file path from `themes/hugo-coder/layouts/...` into the project's top-level `layouts/...`; Hugo's lookup order makes the project copy win.
- `assets/` holds image files used by the site (referenced by Hugo's asset pipeline / image processing, not by raw URL).
- `motion.html` and `picture.html` at the repo root are leftover standalone HTML pages predating the Hugo migration. They are not part of the Hugo build. Do not edit them expecting them to appear on the site.
- `public/` and `resources/` are build artifacts. Do not hand-edit; they are regenerated by `hugo`.
- `.hugo_build.lock` is a transient lockfile from a running `hugo` process; safe to delete if stale.

## Deployment

GitHub Pages serves the site at `https://skhlv.com` via `.github/workflows/hugo.yml`. The workflow runs on push to `master` (and via manual `workflow_dispatch`), builds with the pinned `HUGO_VERSION`, and deploys with `actions/deploy-pages@v4`. The custom-domain binding is kept in `static/CNAME` so Hugo copies it into every build — do not move it back to the repo root or the binding can drop on deploy.

To bump Hugo, update `HUGO_VERSION` in the workflow to match the locally tested version.
